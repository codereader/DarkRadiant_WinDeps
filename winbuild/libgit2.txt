== Notes about the libgit2 folder ==

- Download the sources for libgit2 and extract them. 
- Open a Visual Studio command prompt, change to that folder
- cmake . -DBUILD_CLAR=OFF
- Open the libgit2.sln Solution File 
- Remove the libgit2_clar and RUN_TESTS projects
- Move the include/ folder to windeps/windeps/libgit2/include/
- Change the Project settings in git2internal:
- => Additional include folders: $(SolutionDir)..\..\windeps\libgit2\include
- Change the output name settings of the git2 project
- => Release Build output name: libgit2
- => Debug Build output name: libgit2-d
- => Linker Settings > Advanced > Import Library: $(OutDir)$(TargetName).lib
- => Linker Settings > Debugging > Generate Program Database File: $(OutDir)$(TargetName).pdb
- => Linker Settings > Debugging > Generate Debug Info: Yes /DEBUG
- Remove the folders CMakeFiles, azure-pipelines, .github, fuzzers, examples, docs
- Add git2 project Post-Build Event
mkdir $(SolutionDir)..\..\w64deps\libgit2\bin
mkdir $(SolutionDir)..\..\w64deps\libgit2\lib
copy /Y $(TargetPath) $(SolutionDir)..\..\w64deps\libgit2\bin
copy /Y $(OutDir)$(TargetName).pdb $(SolutionDir)..\..\w64deps\libgit2\bin
copy /Y $(OutDir)$(TargetName).lib $(SolutionDir)..\..\w64deps\libgit2\lib
